AWS App Mesh¶

概要¶

AWS App Meshは、Amazon Web Services (AWS)が提供するマイクロサービス用のサービスメッシュであり、アプリケーションレベルのネットワークトラフィックの制御と監視を可能にします。

サービスメッシュとは、マイクロサービス間の通信を透過的に、かつ統一的に管理・制御するためのインフラストラクチャの一つで、これにより各マイクロサービスがそれぞれ通信制御のためのロジックを持つ必要がなくなります。App MeshはこのサービスメッシュをAWS環境で容易に構築・運用できるようにするサービスです。

仮想化と抽象化¶

  • ベアメタルサーバー：サーバーハードウェア、OS、ライブラリ、アプリケーションから構成されており、使用率に関わらずハードウェアのコストは変わらない

  • 仮想マシン：OSの上に仮想化プラットフォーム上にゲストOSを構築しているが、仮想化レイヤが重いことがデメリットである

主な特徴¶

  • アプリケーションレベルのネットワーク制御: 各マイクロサービスがそれぞれ通信制御のためのロジックを持つ必要がなく、一貫性と可視性を確保します。

  • トラフィックルーティング: フレキシブルなトラフィックルーティングルールを定義して、サービス間の通信を制御できます。

  • セキュリティ強化: サービス間の通信をTLSで暗号化し、セキュリティを強化します。

  • AWSサービスとの統合: Amazon ECS, Amazon EKS, Amazon EC2、およびAWS FargateなどのAWSのコンピューティングサービスと統合されています。AWS X-Rayとの統合もあり、アプリケーションのパフォーマンス情報を収集し、エンドツーエンドのトレースを視覚化することも可能です。

主な用途¶

  • マイクロサービスアーキテクチャの通信制御: マイクロサービス間の通信を透過的に、かつ統一的に制御します。

  • 運用の自動化: リリースやローリングアップデート、キャニャリーリリースなどの運用を自動化します。

  • セキュリティ向上: サービス間の通信を暗号化し、セキュリティを強化します。
